Release Notes - PROTOMAK - Version 0.0.1.RELEASE - 22nd June 2011

Bug

  • [PMK-19] - Package doesn't get written for anonymous simple types
  • [PMK-23] - Attributes are currently returned in sparse order and therefore indexes are not consistent.

Epic

  • [PMK-1] - As a user I'd like to have an API which converts XSDs into proto files
  • [PMK-24] - Create a Protomak Maven Plugin
  • [PMK-25] - Create an Ant Task for Protomak

Story

  • [PMK-4] - Make Protomak backwards compatible with Java 5
  • [PMK-5] - Create an XSD which reflects a Proto file structure.
  • [PMK-6] - Create protomak-engine Website documentation
  • [PMK-7] - Setup infrastructure for XSD parser in Maven
  • [PMK-8] - As a user I'd like a service which fills a Proto domain model from an XSD
  • [PMK-9] - as a developer, i'd like to obtain line manager and compliance approval to work on this project.
  • [PMK-10] - Write a service which gets as input Protomak proto domain model (a ProtoType) and writes a .proto file
  • [PMK-14] - Provide a comprehensive set of XSD schemas that Protomak should support.

Sub-task

  • [PMK-11] - Resolve use case when XSD has got a single element
  • [PMK-12] - Resolve use case when XSD has got more than a single element but it hasn't got complex types
  • [PMK-13] - Resolve use case when XSD has got a single element of type complex
  • [PMK-15] - Create a method which manufactures a package name from a target namespace
  • [PMK-16] - Create an utility method which, given a min and max occurs as returned by XSOM it returns a MessageAttributeOptionalType with the correct optionality value
  • [PMK-17] - Support simple and complex types in the same file. For SimpleTypes, support restrictions
  • [PMK-18] - Create support for anonymous types
  • [PMK-21] - Create support for extensions
  • [PMK-22] - Ensure that the order of processing multiple complex types and elements is deterministic
  • [PMK-26] - Ensure that generated proto files can be converted to java objects
